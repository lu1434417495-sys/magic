# 战斗单位贴图要求

## 关联上下文单元

- CU-18：战斗展示主链
- CU-19：自动化回归与截图辅助
- CU-20：敌方模板、AI brain、行动定义种子内容

当前实现边界以 [`project_context_units.md`](project_context_units.md) 为准；本文只定义战斗棋盘单位贴图要求，不替代 battle renderer 或敌人内容真相源。

## 目标

当前战斗棋盘已经能正确显示地形，但人物和怪物在地图上的辨识度不足。问题不在世界地图大图质量，而在于战斗棋盘需要一套单独的、按棋盘尺寸和锚点设计的贴图。

这份文档只定义战斗棋盘单位贴图要求，目标是让玩家在默认镜头下能一眼看清：

- 单位站在哪个格子
- 是我方还是敌方
- 是人形小体型还是大体型怪物

## 现有运行时约束

- 战斗地格是 `64x32` 的等轴菱形。
- 默认镜头缩放是 `2.0x`。
- 单位当前按“脚底锚点”贴到地格中心附近，而不是按整张图左上角对齐。
- `1x1` 单位占单格。
- `body_size >= 3` 的大体型单位当前占 `2x2`。
- 单位层启用了 `y_sort`，所以贴图底部接地点必须清晰准确。

## 你需要绘制的资产

第一批只需要做战斗棋盘贴图：

- 普通人物 / 小怪：`1x1` 棋盘战斗贴图
- 大怪 / Boss：`2x2` 棋盘战斗贴图

第二批可选：

- HUD 头像图

## 一、棋盘战斗贴图

### 1. 通用要求

- 格式：`PNG`
- 背景：完全透明
- 帧数：先做单帧待机图，不做动画条
- 视角：`3/4` 站姿，偏向镜头，不要纯正面、纯侧面、纯俯视
- 构图：重点突出头部、肩线、武器或翅膀等识别特征
- 不要带方形背景、雾墙背景、地台背景、光圈底板
- 可以有非常轻的接地阴影，但必须很弱，不能画成黑色大底盘

### 2. 1x1 普通单位

- 建议画布：`96x128`
- 锚点：`(48, 112)`，即底部中心偏上，作为脚底接地点
- 实际接地宽度：`20-28 px`
- 主体可见高度：`72-96 px`
- 左右最大外扩：相对锚点不超过 `32 px`
- 底部保留：锚点下方保留 `8-16 px` 空间给透明边缘或轻阴影

适用对象：

- 主角
- 人类敌兵
- 骷髅、盗匪、术士、狼这类单格怪

### 3. 2x2 大体型怪物

- 建议画布：`192x160`
- 锚点：`(96, 136)`，即整张图底部中心，代表 `2x2` footprint 的重心接地点
- 实际接地宽度：`64-96 px`
- 主体可见高度：`96-132 px`
- 允许头、角、翅膀、尾巴超出 footprint，但最低接地点不能跑到 footprint 外
- 不允许把四肢最低落点画到“看起来踩在 2x2 外侧格子”的位置

适用对象：

- 龙
- 巨兽
- 大型恶魔
- 明显占两格以上视觉体量的 Boss

### 4. 方向和重心要求

- 所有单位默认采用统一朝向，不要每张图朝向完全不同。
- 推荐朝向：面向镜头的 `3/4` 视角，允许轻微偏向屏幕右侧。
- 角色重心要落在锚点正上方，不要做成身体偏左、脚底偏右这种“站不稳”的构图。
- 长武器、披风、尾巴可以外扩，但它们不能替代脚底成为视觉主接地点。

### 5. 可读性要求

这是最重要的部分。战斗棋盘的地表现在以棕、灰、土黄为主，如果贴图也全是暗棕和黑灰，会直接糊成一团。

- 头部、肩膀、武器必须形成清晰轮廓
- 角色上半身必须有一块中亮区域，不接受整张图都是暗色团块
- 不要把大量烟雾、碎屑、火花画到主体外围，避免和地表贴图打架
- 不要把脚边做成非常厚重的黑影，会让站位边界变脏
- 玩家单位建议带冷色或亮色识别点
- 敌方单位建议带暖色、红色、橙色或毒绿色识别点
- 同一阵营内部可以同风格，但每个单位轮廓必须能一眼区分

最低验收标准：

- 缩到 `96x128` 或 `192x160` 的最终画布后，头、武器、肩线仍能看清
- 放到棕色峡谷地表上，不会和地表颜色融成一块
- 只看剪影也能大致分出“战士 / 法师 / 兽 / 龙”

### 6. 不建议直接复用世界地图大图

项目里现有 `assets/main/basic_map/npc/` 更像角色展示图或世界地图大图，细节很多、边缘烟雾很多、画面占比也偏大。

这些图不适合直接缩成战斗棋盘贴图，原因是：

- 缩小后细节会糊
- 外围烟雾会污染地格边界
- 主体重心不一定对准棋盘锚点
- 很多图是展示构图，不是站位构图

正确做法是：

- 保留现有角色设定和材质风格
- 重新做一张“战斗棋盘版”
- 重点强化轮廓、头肩、武器和阵营识别色

## 二、文件命名和目录建议

建议单独放一套战斗单位资源，不和世界地图 NPC 图混用。

建议目录：

- `assets/main/battle/units/player/`
- `assets/main/battle/units/enemy/`
- `assets/main/battle/portraits/`

建议命名：

- `assets/main/battle/units/player/<unit_id>_board.png`
- `assets/main/battle/units/enemy/<unit_id>_board.png`
- `assets/main/battle/units/enemy/<unit_id>_board_2x2.png`
- `assets/main/battle/portraits/<portrait_id>.png`

如果你希望我后面直接接运行时加载，`unit_id` 最好和战斗数据里的 `unit_id` 或 `portrait_id` 一致。

## 三、HUD 头像图，可选

这部分不是当前最大问题，但如果你顺手要一起画，建议单独出一套，不和棋盘图复用。

- 格式：`PNG`
- 建议尺寸：`512x512`
- 构图：胸像或半身像
- 背景：透明或极简纯色，不要复杂场景
- 重点：脸、头盔、角、眼睛、职业特征

用途：

- 战斗界面左下单位卡
- 行动队列头像

## 四、给画师的硬性红线

- 不要交 `768x768` 展示图然后让我运行时强缩
- 不要带大面积白雾、黑烟、地面、石台、光晕背景
- 不要把脚底画偏
- 不要让最暗区域和峡谷地表颜色完全重合
- 不要把 2x2 Boss 画成“看起来踩了 3x3”
- 不要在贴图里烘焙选中框、血条、阵营圈

## 五、交付建议

建议你先给我一轮最小样本，我可以直接在战斗棋盘里替换验证：

- 我方近战 `1x1` 1 张
- 我方法系 `1x1` 1 张
- 敌方人形 `1x1` 1 张
- 敌方兽类 `1x1` 1 张
- 大体型怪物 `2x2` 1 张

这一轮只要站位、轮廓和清晰度对了，后面整批替换会很快。
