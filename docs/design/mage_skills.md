# 法师技能设计文档

更新日期：`2026-04-13`

说明：
- 本文档只补充设计方案，不涉及代码、资源或数值表落地。
- 命名尽量与现有仓库中的 `mage_fireball`、`mage_ice_lance`、`mage_chain_lightning`、`mage_arcane_mind_i`、`mage_spellthread_i`、`mage_mana_reservoir_i` 保持同一套语义。
- 统一按 `射程 / 范围 / 命中 / 伤害 / 消耗 / 冷却 / 效果` 描述，便于后续挑选实现。
- 默认伤害按 `magic_attack / magic_defense / 对应元素抗性` 结算；当技能描述为“法阵 / 墙 / 领域 / 召唤 / 地面落点”时，默认建议走 `target_mode = ground` 或 `ground + cast_variants`。

---

## 一、职业定位

- 核心身份：远程法术炮台、元素控场者、地形改写者。
- 战斗节奏：先用减速、致盲、感电、泥沼和法阵重塑局势，再用高阶塑能法术收割关键目标。
- 资源建议：常规技能以 `mp` 为主；高阶领域技、召唤技和终结技使用 `aura`。
- 数值倾向：法师不应依赖厚血硬吃回合，而是用区域收益、状态联动和施法距离换取战场控制权。

---

## 二、复用状态与联动关键词

- 直接复用现有状态：`frozen`、`shocked`、`blinded`、`staggered`、`armor_break`、`mud`。
- `灼烧`：回合结束受到一次火焰法术伤害，持续 `2` 回合；若目标原本处于 `frozen`，则先移除 `frozen` 再触发一次额外爆燃伤害。
- `寒蚀`：移动力 `-1`，持续 `2` 回合；目标承受下一次物理或穿刺伤害时额外受到 `+10%` 伤害。
- `浸湿`：闪电抗性 `-15%`，且更容易被施加 `frozen`；与 `mud`、水面或暴雨类地形联动良好。
- `感电`：命中 `-10%`，持续 `1` 回合；下一个雷系技能对其追加 `+15%` 伤害或 `+1` 连锁半径。
- `奥脆`：魔法防御 `-15%`，持续 `2` 回合；适合作为奥术、诅咒和队友法术的共通破口。
- `咏唱`：自身下一个法术射程 `+1`、命中 `+10%`；若本回合未移动，再额外 `+10%` 伤害。
- `法盾`：优先吸收一次法术或远程伤害，数值可按技能强度配置。
- `虹吸`：攻击带有该状态的目标时回复少量 `mp`；适合死灵和奥术持续战。

---

## 三、与现有技能体系的映射

- 技能 ID 统一使用 `mage_` 前缀，便于直接挂入 `SkillDef` 与 `ProfessionDef`。
- 常规单体法术优先落在 `single`，高频 AOE 优先落在 `radius 1/2`，穿透技优先落在 `line`，近身压制优先落在 `cone` 或 `self`。
- 地形、法阵、墙、召唤物技能优先走 `ground` 或 `cast_variants`，避免为单一技能额外写特判。
- 常规输出使用 `mp`，少数“领域 / 大型召唤 / 终结禁咒”使用 `aura`，和 `skills_implementation_plan.md` 中的资源规划一致。
- 法师的单体技能要能和弓箭手的 `标记`、战士的 `armor_break`、泥沼地形、`shocked / frozen / blinded` 共用收益，而不是做一套封闭数值。

## 五、设计原则

- 法师的强度应建立在“区域塑形 + 状态联动 + 资源管理”上，而不是把所有高值都堆进单发倍率。
- 火、冰、雷、土、奥术、幻术、召唤、死灵八条分支都要能独立成型，但也必须存在跨分支连段空间。
- 地面技与持续领域技应优先复用 `ground`、`cast_variants`、`terrain_replace`、`height_delta` 等已有设计方向，不单独造协议。
- 法师控制不应直接替代专职硬控职业，更适合做“减速、致盲、断施法、拆阵型、放大后续伤害”。
- 高阶 `aura` 技能应是战斗转折点，而不是法师每回合都能稳定打出的常规动作。
- 召唤与防护必须给出明确机会成本，避免把法师变成既最安全又最高伤的全能职业。
