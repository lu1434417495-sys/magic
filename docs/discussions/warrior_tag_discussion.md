# Warrior 职业 Tag 讨论结论

更新日期：`2026-04-23`

## 状态

- 当前状态：`Active Discussion Record`
- 范围：仅记录本轮关于 `warrior` 与近战共通被动 tag 的讨论结论。
- 说明：本文件是设计讨论纪要，不代表代码与数据已完成落地。

## 本轮结论

### 1. `warrior` 采用两层 tag，而不是高阶门槛

- 不拆“高阶门槛”层。
- 当前只保留两层：`职业门槛`、`通用表现`。

### 2. `warrior` 的职业门槛 tag

- `近战武器`
- `突进`
- `武装防御`

### 3. `warrior` 的通用表现 tag

- `防御`
- `控制`
- `范围`
- `位移`
- `输出`
- `支援`
- `恢复`
- 需要更细时，可继续补充：`连击`、`斗气`、`破防`、`战吼`、`仇恨牵制`、`反击`、`被动`、`受击触发`、`命中增伤`。

## 关键约束

### 1. `武装防御` 不等于通用 `防御`

- `武装防御` 用来表达战士系、近战武装姿态、防反、格挡、盾击这一类职业门槛。
- 通用 `防御` 只是技能表现描述，法师的护盾、减伤、法术防御都可以挂 `防御`，但不应该因此计入 `warrior` 的职业成长。

### 2. `突进` 不等于通用 `位移`

- `突进` 只用于职业门槛层，表达战士靠冲锋、贴身切入、起手逼近的能力。
- 普通位移、跳跃、调整站位只应先视为 `位移` 表现，不必自动算作 `突进`。

### 3. 不是所有战士技能都必须贡献职业门槛

- 有些技能可以只挂通用表现 tag。
- 例如 `战吼`、`嘲讽` 这类技能更像行为表现，不应为了凑职业门槛而强行挂到 `武装防御`。

### 4. 共享近战被动不要绑死在 `warrior`

- 近战职业共享技能优先走“近战共通”语义，而不是 `warrior` 专属语义。
- 这类技能可使用：`近战武器`、`近战反制`、`武器精通`。
- 这样更容易复用到 `rogue`、未来的 `剑圣` 等职业。

### 被动技能

| 技能类型 | 建议职业门槛 | 建议通用表现 | 说明 |
| --- | --- | --- | --- |
| `顺势斩` 这类追击/补刀被动 | `近战武器` | `输出`、`连击` | 只有在明确依赖冲锋切入时，才追加 `突进` |
| 受击后消耗反射点反击 | `近战武器` 或 `近战反制` | `反击`、`被动`、`受击触发` | 适合战士、盗贼、未来剑圣等共用，不建议挂 `武装防御` |
| `武器精通` | `近战武器`，必要时补 `武器精通` 作为共通主轴 | `被动`、`命中增伤` | 若后续形成整条共享天赋链，建议把 `武器精通` 升格为显式共通门槛 |

## 当前建议的判定规则

### 职业门槛层

- 只用于职业成长、转职判定、职业主题归属。
- 必须保持语义窄、边界清楚。
- 当前 `warrior` 只认：`近战武器`、`突进`、`武装防御`。

### 通用表现层

- 用于技能检索、UI 展示、AI 识别、内容统计、平衡分析。
- 可以跨职业复用。
- 不应直接触发 `warrior` 升级或转职条件。

## 暂留开放项

- `近战反制` 是否正式设为独立的近战共通门槛。
- `武器精通` 目前先视为“可升格”的近战共通主轴，是否单独成为门槛 tag，取决于后续是否围绕它继续扩展技能树。
- 其他职业的职业门槛 tag 还未确认，后续应按同样方法逐个拆分，避免互相污染。
